### Vibe Coding
- **바이브 코딩**(Vibe Coding)은 2025년 2월 OpenAI 공동 창립자인 Andrej Karpathy가 처음 제시한 개념으로, 개발자가 AI 서비스의 도움을 받아 자연어로 코드를 작성하는 새로운 프로그래밍 방식을 의미
- 사전에 엄밀한 논리나 설계보다는 직감과 느낌에 의존하여 코드를 생성한다고 하여 '바이브' 코딩이라는 이름이 붙음

### MCP(Model Context Protocol)
- **MCP**(Model Context Protocol)는 2024년 11월 Anthropic이 도입한 개방형 표준 프로토콜로, AI 모델이 외부 데이터 소스 및 도구와 상호작용하고, 문맥을 더 정확하게 이해하며, 상황에 맞는 작업을 수행할 수 있도록 돕는 기술

### Netlify
- **Netlify**는 웹 개발자를 위한 클라우드 기반의 플랫폼으로, 웹사이트와 웹 애플리케이션의 배포, 호스팅, CI/CD(지속적 통합 및 배포), 서버리스 기능 등을 통합적으로 제공
- 특히 정적 사이트 배포에 최적화되어 있으며, JAMstack(JavaScript, API, Markup) 아키텍처를 기반으로 한 프로젝트에 강력한 기능을 제공

### Gemini CLI
- **Gemini CLI**는 구글의 Gemini AI 모델을 터미널에서 직접 활용하도록 설계된 명령줄 인터페이스 도구
- 텍스트 생성, 코드 작성, 아이디어 구상 등 다양한 AI 작업을 터미널 명령어 입력만으로 빠르고 효율적으로 수행할 수 있음

### On-premise
- **온프레미스(On-Premises)**란, 말 그대로 자체 보유한 서버나 데이터센터에서 직접 하드웨어와 소프트웨어를 설치·운영하는 방식
- 즉, 클라우드(AWS, GCP, Azure 등) 같은 외부 서비스에 의존하지 않고, 회사 내부 서버실이나 IDC에 물리적인 장비를 두고 그 위에 시스템을 운영하는 환경

### OKLab
- perceptual uniform을 목표로 만들어진 색 공간
- 즉, 수치상의 차이가 실제 인간이 느끼는 색 차이와 최대한 일치하도록 설계된 것

### OKLch
-  OKLab을 **극좌표(polar coordinates)**로 변환한 색 공간.
-  L : Lightness(밝기)
-  C : Chroma(채도)
-  H : Hue (색상각, 0°–360°)

### Retrieval-Augmented Generation(RAG)
- 대규모 언어 모델(LLM)에 외부검색+조회능력을 더해 생성을 강화하는 방식
- 더 정확하고 최신의 답변을 내는 구조

### Vision-Language Model(VLM)
- 시각(이미지,영상)과 언어(텍스트)를 함께 이해,처리하는 모델
- 비전 인코더, 언어모델 멀티모달 결합 모듈로 이루어짐

### 애자일 - 스크럼 방법론
- 1~4주 단위의 짧은 주기 안에서 팀이 자율적으로 계획, 개발하고 피드백을 받아 제품을 지속적으로 발전시키는 프레임워크

### 인프라 온 코드(Infrastructure as Code, IaC)
- 인프라를 코드로 정의하여, 버전 관리와 자동화가 가능한 형태로 운영하는 방식
- 서버, 네트워크, 방화벽, 로드밸런서 등을 코드로 구성·배포
- Terraform
- Pulumi
- AWS CloudFormation
- Azure Resource Manager
- Google Cloud Deployment Manager

### GPT-SoVITS
- GPT-SoVITS = GPT 기반 텍스트 모델 + SoVITS 음성 합성 모델
- GPT: 텍스트(또는 음소 시퀀스)를 보고 말의 흐름, 억양, 리듬을 예측
- SoVITS:예측된 정보와 화자 음색을 결합해 실제 음성 파형을 생성
